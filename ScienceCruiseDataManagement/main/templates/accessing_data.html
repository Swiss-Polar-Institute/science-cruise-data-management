Accessing ACE data

Introduction

All of the data produced during the cruise (by scientists and by the ship) are being stored on central servers, so that they are safely backed up and can be stored at the Swiss Polar Institute on a long-term basis after the cruise.

In addition to this, some supporting data from external sources such as weather forecasts and bathymetry data are also available FOR USE DURING THE CRUISE. Note that a lot of these additional datasets have strict licenses that must be adhered to.

You are able to access all of these data by following some simple steps in the section, Data Access, below.

Data areas

Below are descriptions of the different data areas:

ace_data - read only - all data collected on board the cruise are stored and backed up here. Organised by instrument.

external_data - read only - data from external sources that is of interest to parties on board. Includes terrain maps, satellite images and bathymetry of the islands we will visit.

intranet_documents - read only - a collection of useful and interesting documents. If you would like anything adding here, please speak to the Data Management team.

ship_data - read only - data streams that come from ship-based instruments: navigation (GPS), weather, multibeam, ADCP. Some of these are still to be added (26/12/2016).

work_leg1 - read and write - please use this area to save things on which you are working that are related to the cruise, and that you would like backed up. Please put things in a folder named after your project, eg. project11, so we know to whom it belongs. THIS FOLDER WILL BE STORED PERMANENTLY FOLLOWING THE CRUISE SO PLEASE CONSIDER CAREFULLY WHAT YOU STORE HERE. This area is also accessible by everyone so do not store any personal information here. If you would like to share your photos, please create a folder with your full name as the title and put this inside the photos directory.

Data access

Connect to the ship's network using an ethernet cable. All cabins are connected to the network, as are some labs. If for some reason you find an ethernet port does not work, please report it to the Data Management team with the room and port number, i.e. Room 336, port XS30/BT.

You should now have access to the ACE intranet as well as the data storage area.

Ubuntu

NAS host name: storage.lan

Come and see the Data Management team if you have problems - each computer will be different.

Windows

Click on start and run, then type: \\storage.lan - this will take you to the set of folders.

Mac

Click on go then connect to server.

Type smb://storage.lan and you can add this to a list of favourites by clicking on the + button next to where you enter this address. Seleect whichever area you need and double click: it will open in the file browser.

**If you need any help please pop along to the Expedition Office at any time. We sometimes have chocolates....**

External data - a little more informtion

Satellite data

Sea surface temperature (SST), chlorophyll (chl) and sea-ice data are provided on a daily basis.

Data sources: Oceancolor (NASA), MODIS (sent by Rafel Sim√≥ - thanks Rafel!)

Bathymetry and terrain maps of islands

You will find a set of bathymetry data, satellite images and terrain maps for the island work. These are generally sorted by the island names, but for larger providers, there could be data for different areas within the provider name (eg. GEBCO). A lot of these datasets come with license agreements and citations which you will hopefully find within the relevant folders in txt or html files. PLEASE observe these licenses and obtain your own copy of the data when back on dry land if necessary.

Ship data - a little more information

Meteorological data

The ship's meteorological data can be found here in daily text files. Each file contains a set of data that are recorded every 30 seconds, and another set that are recorded every 3 seconds. The headers are different so be aware if trying to parse the files. They will appear in the database very soon and other people on board have already parsed the files so please ask before doing it yourself if you want to save some time.

Navigation (GPS) data

Two sources of GPS data are being brought into this data area in daily files which are updated live. These data are already stored in the database and can be fed into some mapping software so you can see the position of the ship live. The data are collected with a 1-second resolution but the data management team can easily output this in a different resolution if required, just ask. Finally, at the end of each leg, a JPEG of the track and a csv file with positions will be provided in the intranet documents section.
